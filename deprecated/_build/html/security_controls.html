Security Controls¶

A security control is any safeguard or countermeasure that minimizes the risk of security incident. There are technical and non-technical controls. The latter are outside the scope of this document.

NIST Special Publication 800-53¶

In Revision 4 of the Security and Privacy Controls for Federal Information Systems and Organizations, security controls are broken down into “Control Families.” The following sections detail control families with technical controls that can be implemeted by OpenShift and its certified images.

Security Control Traceability Matrix¶

SCTMs are broken down by control families. Security controls which are not implemented by the OpenShift ecosystem are unlisted.

  1. SCTM for Access Control
  2. SCTM for Awareness and Training
  3. SCTM for Audit and Accountability
  4. SCTM for Change Management
  5. SCTM for Contingency Planning
  6. SCTM for Identification and Authentication
  7. SCTM for Incident Response
  8. SCTM for Maintenance
  9. SCTM for Media Protection
  10. SCTM for Personnel Security
  11. SCTM for Planning
  12. SCTM for Security Assessment and Authorization
  13. SCTM for Physical and Environment Protection
  14. SCTM for System and Communications Protection
  15. SCTM for System and Information Integrity
  16. SCTM for System and Services Acquisition
